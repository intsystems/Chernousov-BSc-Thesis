# Вычислительные эксперименты

Код к работе о мере сложности выборки $\mu_D(X) = \log\det(I + K)$,
где $K_{ij} = \exp(-\lambda\, D^2(x_i, x_j))$, а расстояние $D$ вычисляется
между вложениями объектов, построенными по предобученной диффузионной модели.
В коде мера обозначается `C(X)` (в тексте работы — $\mu_D(X)$, собственные
числа — $\nu_k$).

## Установка

```bash
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

`torch`, `torchvision`, `diffusers` нужны только для exp4; exp1–exp3 работают
на чистых `numpy`/`scipy`/`matplotlib`.

## Структура

```
core/                   общая библиотека (эксперименты её используют, но не меняют)
├── model.py            расписания шума (VE/VP), аналитический скор гауссовой смеси
├── complexity.py       конвейер: вложения → расстояния → ядро → C(X) = log det(I + K)
└── trajectory.py       интегрирование PF-ODE, траекторное вложение, веса Beta(a, b)

exp1_algebraic/         алгебраические свойства меры
exp2_gmm/               скор-вложение на гауссовых смесях (аналитический скор)
exp3_ode_trajectory/    траекторное вложение (аналитический скор, VP)
exp4_mnist/             MNIST с предобученной DDPM
```

Результаты каждого эксперимента пишутся в его `results/` (не попадает в
репозиторий); графики, используемые в тексте или слайдах, копируются в
`figures/` в корне репозитория.

## Соглашение о выборе λ

`compute_complexity` требует явный `lambda_`: ширина ядра — свойство модели,
а не измеряемой выборки. λ калибруется один раз медианной эвристикой по
**независимой референсной выборке** (`calibrate_bandwidth`) и переиспользуется
для всех измерений при данной модели. Подбор λ по самой измеряемой выборке
нормализует структурные различия между выборками и запрещён конвейером
(диагностическая иллюстрация — п. 2.2 ноутбука exp2).

## Эксперименты

### exp1 — алгебраические свойства (без модели)

Прямая проверка всех доказанных свойств меры на синтетических вложениях:
перестановочная инвариантность, ограниченная монотонность
$\log\frac{N+2}{N+1} \le \Delta \le \log 2$, субаддитивность, субмодулярность,
асимптотики (N одинаковых точек, M кластеров, максимально разнообразные точки),
PSD ядра, отрицательная определённость $D^2$, липшицева устойчивость.

```bash
cd exp1_algebraic
python run.py                 # 89 проверок, код возврата 0 при успехе
python plot_asymptotics.py    # графики асимптотик → results/
```

### exp2 — скор-вложение на гауссовых смесях (ноутбук)

`exp2_gmm/experiment.ipynb`: валидация меры со **скор-вложением**
$\Phi(x) = (s(x, t_1), \dots, s(x, t_L))$ и точным аналитическим скором.
Зависимость $C$ от числа компонент, расстояния между ними, дисперсии, $N$, $D$;
чувствительность к гиперпараметрам $\lambda$ и $L$; спектр ядра; фиксированная
модель на разных выборках. Главный вывод: скор локален, поэтому хорошо
разделённые моды почти не различаются скор-вложением — это мотивирует
траекторное вложение exp3.

### exp3 — траекторное вложение (скрипты + ноутбук)

Вложение $\Phi(x) = z_x(\cdot)$ — траектория PF-ODE, выпущенная из $x$;
скор по-прежнему аналитический (VP-расписание).

```bash
cd exp3_ode_trajectory
python run_growth.py          # скорость роста: N кластеров ~ N log 2, один кластер ~ log(1+N)
python run_separation.py      # C от расстояния между кластерами (λ от одного кластера)
python run_manifold_dim.py    # C от N при разной внутренней размерности данных
python run_linear_kernel.py   # гауссово ядро против линейного, включая D=100
```

`results/growth_rate.png` из `run_growth.py` используется в слайдах
(`figures/growth_rate.png`). `experiment.ipynb` — визуализация траекторий и
те же эффекты в ноутбучном виде.

### exp4 — MNIST с предобученной DDPM

Модель `1aurent/ddpm-mnist` (HuggingFace, загружается автоматически).
Траектории PF-ODE интегрируются батчевым методом Эйлера
(`pretrained.batched_ode_trajectories`) — поточечный адаптивный RK45 из
`core/` для нейросетевого скора непрактичен.

```bash
cd exp4_mnist
python run_growth.py          # C(X) от N: один класс против смеси классов
python run_growth.py --quick  # сокращённый дымовой прогон (CPU)
python export_checkpoint.py   # выгрузка чекпойнта для Colab-ноутбуков
```

Ноутбуки `trajectory_colab.ipynb` и `perceptual_colab.ipynb` — автономные
Colab-версии (GPU): первый повторяет конвейер exp4 без импорта `core/`,
второй — поисковый вариант с «перцептивным» расстоянием через расхождение
предсказаний шума. Обоим нужен файл `ddpm_mnist_pretrained.pt`
из `export_checkpoint.py`.

## Воспроизводимость

Во всех экспериментах фиксированы сиды (`SEED = 42` для измеряемых выборок,
`99` — для калибровочных). Скрипты детерминированы: повторный запуск
воспроизводит графики байт-в-байт (exp1–exp3, CPU).
